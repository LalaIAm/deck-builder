# Consolidated Learnings

## Memory Bank Initialization Pattern

- When onboarding a repository with partial memory docs, initialize the full core Memory
  Bank set in one pass: project brief, product context, active context, system patterns,
  tech context, and progress.
- Preserve existing repository naming conventions for memory files when they already
  exist (e.g., `projectbrief.md`), to avoid breaking local workflows or open editor tabs.
- Capture immediate next steps and known risks during initialization so the next session
  can start execution without re-discovery.
