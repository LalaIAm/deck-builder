# Product Context

## Why This Project Exists

Creating a full tarot deck traditionally requires significant creative labor, budget,
and iteration cycles. This project exists to make complete deck production accessible
through an autonomous pipeline that can maintain visual consistency across all 78 cards.

## Problem It Solves

- Manual, fragmented card-by-card generation workflows
- Inconsistent style, symbolism, and quality across cards
- Time-consuming QA and rework loops
- Lack of a structured process for autonomous end-to-end generation

## Target Outcome

Deliver a commercially viable, production-ready tarot deck where each card aligns to a
shared style system and quality bar, without requiring manual intervention for each card.

## User Experience Goals

- One-command (or minimal-input) generation flow
- Deterministic, inspectable outputs (manifest + structured scores)
- Automatic issue detection and correction via retries
- Clear artifact trail for each card (concept, prompt, evaluation, final image)
